# anchor-template — Decision Log

Green-field single anchor (INR ↔ USDC) on the official Stellar Anchor Platform
image. Separate from `../anchor-service/` (the parked factory). Numbering is local
to this project.

---

## AT-001 — Green-field, not extending anchor-service
The canonical anchor is a fresh project using `stellar/anchor-platform:latest`
(v4.4.0), reusing proven patterns from `anchor-service/` (callback contract,
SEP-24 interactive, Platform API PATCH) as reference only — not its factory
entanglement or ANCH mint logic.

## AT-002 — Asset model: real Circle USDC (treasury/float), not mint
The anchor does not issue USDC; it holds a USDC **float** in a treasury account
(the AP asset `distribution_account`) and transfers real USDC. This makes treasury
management and INR/USD FX (SEP-38 `/rate`) first-class. Testnet USDC issuer:
`GBBD47IF6LWK7P7MDEVSCWR7DPUWV3NY3DTQEVFL4NAT4AQH3ZLLFLA5` (Circle) — kept in
config; testnet resets quarterly, so re-verify after a reset.

## AT-003 — AP v4.4.0 callbacks are only /customer and /rate
Verified against the upstream source: the `callback/` package has only
`RestCustomerIntegration` (SEP-12) and `RestRateIntegration` (SEP-38). There is
**no `/unique_address` or `/fee` callback** in this version — those routes in
`anchor-service` are dead code. Our business-server exposes only `/customer` and
`/rate` (at the callback_api base_url root), keeping the contract clean.

## AT-004 — Two keypairs, generated by setup-testnet.mjs
`signing` (SEP-10 home-domain signing key) and `treasury` (USDC float). No issuer
keypair (Circle issues USDC). Setup funds both with XLM via Friendbot and adds a
USDC trustline to the treasury; funding the USDC float itself is Phase B (no
Friendbot for USDC). Config (`stellar.toml`, `assets.yaml`) is rendered from
`*.template` files; secrets go to `.env` (git-ignored).

## AT-005 — Host ports are env-overridable (canonical defaults 5432/3000)
`DB_HOST_PORT` / `BIZ_HOST_PORT` let the stack coexist with other local stacks
without changing the canonical template defaults. The SEP-24 `interactive_url`
base_url assumes business-server on 3000 — change both together for a real wallet
flow on a remapped port.
